## Criar Pagamento PIX

Este endpoint permite gerar um novo QR Code PIX para recebimento.

### Requisição

<ParamField body="value" type="number" required>
  O valor da transação em Reais (ex: 150.00). Mínimo R$ 1,00.
</ParamField>

<ParamField body="description" type="string">
  Uma descrição curta para a transação que aparecerá no extrato.
</ParamField>

<ParamField body="coverFee" type="boolean" default="false">
  Se `true`, a taxa de R$ 0.50 será descontada do valor recebido. Se `false`, a taxa será cobrada separadamente do saldo da conta.
</ParamField>

<ParamField body="splitUser" type="string">
  E-mail de outro usuário Zynx para divisão de valores (opcional).
</ParamField>

<ParamField body="splitTax" type="number">
  Porcentagem (0.01 a 100) para o `splitUser` (opcional).
</ParamField>

### Resposta

<ResponseField name="success" type="boolean">
  Indica se a operação foi bem-sucedida.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Propriedades do Objeto">
    <ResponseField name="id" type="string">
      ID único da transação gerada.
    </ResponseField>
    <ResponseField name="transactionId" type="string">
      ID interno da transação.
    </ResponseField>
    <ResponseField name="valueInReais" type="number">
      Valor total em reais.
    </ResponseField>
    <ResponseField name="status" type="string">
      Status inicial da transação (geralmente `PENDING`).
    </ResponseField>
    <ResponseField name="qrcodeUrl" type="string">
      URL ou Base64 da imagem do QR Code.
    </ResponseField>
    <ResponseField name="copyPaste" type="string">
      Código PIX Copia e Cola.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash cURL
curl --request POST \
  --url https://api.zynxwallet.com.br/api/v1/payment/create \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: zynx_sua_chave' \
  --data '{
  "value": 150.00,
  "description": "Venda de Produto #123",
  "coverFee": true
}'
```

</RequestExample>

<ResponseExample>

```json Exemplo de Resposta
{
  "success": true,
  "data": {
    "id": "pay_abc123",
    "transactionId": "mistic_xyz789",
    "valueInReais": 150.00,
    "netValue": 14950,
    "fee": 50,
    "status": "PENDING",
    "qrcodeUrl": "data:image/png;base64,...",
    "copyPaste": "00020126360014br.gov.bcb.brcode...",
    "createdAt": "2024-02-16T15:00:00Z"
  }
}
```

</ResponseExample>

## Consultar Pagamento

Verifica o status atual de um pagamento específico.

### Requisição

<ParamField path="id" type="string" required>
  O ID da transação retornado na criação.
</ParamField>

<RequestExample>

```bash cURL
curl --request GET \
  --url https://api.zynxwallet.com.br/api/v1/payment/get/pay_abc123 \
  --header 'X-API-Key: zynx_sua_chave'
```

</RequestExample>

<ResponseExample>

```json Exemplo de Resposta
{
  "success": true,
  "data": {
    "id": "pay_abc123",
    "status": "COMPLETED",
    "value": 15000,
    "netValue": 14950,
    "updatedAt": "2024-02-16T15:05:00Z"
  }
}
```

</ResponseExample>

## Listar Pagamentos

Retorna uma lista paginada de todos os seus pagamentos.

### Requisição

<ParamField query="status" type="string">
  Filtrar por status (`PENDING`, `COMPLETED`, `FAILED`).
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Quantidade por página (máx 1000).
</ParamField>

<RequestExample>

```bash cURL
curl --request GET \
  --url 'https://api.zynxwallet.com.br/api/v1/payment/list?status=COMPLETED&limit=10' \
  --header 'X-API-Key: zynx_sua_chave'
```

</RequestExample>
