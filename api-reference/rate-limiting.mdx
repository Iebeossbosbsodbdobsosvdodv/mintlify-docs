# Rate Limiting

Entenda como funciona o rate limiting da API Zynx Wallet.

## Limites Padrão

Os limites padrão da API Zynx Wallet são:

| Plano | Requisições por Minuto | Requisições por Hora |
|-------|------------------------|----------------------|
| Gratuito | 60 | 1.000 |
| Pro | 300 | 10.000 |
| Enterprise | Customizado | Customizado |

## Headers de Rate Limiting

Cada resposta da API inclui headers que informam sobre seu uso:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1644234567
```

## Tratando Rate Limiting

Quando você atinge o limite, a API retorna um erro 429 (Too Many Requests):

```json
{
  "success": false,
  "error": "rate_limit_exceeded",
  "message": "Você atingiu o limite de requisições.",
  "retry_after": 60
}
```

## Boas Práticas

1. **Cache de Respostas**: Armazene respostas em cache para evitar requisições desnecessárias
2. **Batch Requests**: Agrupe múltiplas operações em uma única requisição quando possível
3. **Monitorar Uso**: Acompanhe seus headers de rate limiting
4. **Implementar Fila**: Para operações em lote, implemente uma fila com delay
