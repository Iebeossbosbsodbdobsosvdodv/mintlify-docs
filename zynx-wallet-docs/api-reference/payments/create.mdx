# Criar Pagamento

Crie um novo pagamento PIX que pode ser pago por seus clientes.

## Endpoint

```
POST /v1/payments/create
```

## Autenticação

Requer autenticação via API Key ou JWT Token. Veja [Autenticação](/getting-started/authentication).

## Parâmetros

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|-------------|-----------|
| `amount` | number | Sim | Valor do pagamento em reais (ex: 100.00) |
| `description` | string | Sim | Descrição do pagamento (máx 255 caracteres) |
| `customer_email` | string | Não | Email do cliente |
| `customer_name` | string | Não | Nome do cliente |
| `customer_phone` | string | Não | Telefone do cliente |
| `expires_in` | number | Não | Tempo de expiração em segundos (padrão: 3600) |
| `metadata` | object | Não | Dados customizados para seu negócio |

## Exemplo de Requisição

### cURL

```bash
curl -X POST https://api.zynxwallet.com.br/v1/payments/create \
  -H "X-API-Key: sua_api_key_aqui" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 150.50,
    "description": "Compra de produto XYZ",
    "customer_email": "cliente@example.com",
    "customer_name": "João Silva",
    "customer_phone": "11999999999",
    "expires_in": 3600,
    "metadata": {
      "order_id": "ORD-12345",
      "product": "Premium Plan"
    }
  }'
```

### JavaScript

```javascript
const apiKey = process.env.ZYNX_API_KEY;

async function createPayment(amount, description, customerEmail) {
  const response = await fetch('https://api.zynxwallet.com.br/v1/payments/create', {
    method: 'POST',
    headers: {
      'X-API-Key': apiKey,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      amount,
      description,
      customer_email: customerEmail,
      expires_in: 3600,
      metadata: {
        timestamp: new Date().toISOString()
      }
    })
  });

  const data = await response.json();
  
  if (!data.success) {
    throw new Error(data.message);
  }

  return data.data;
}

// Usar
const payment = await createPayment(150.50, 'Compra de produto', 'cliente@example.com');
console.log('Pagamento criado:', payment.id);
console.log('QR Code:', payment.qr_code);
```

### Python

```python
import requests
import os

api_key = os.getenv('ZYNX_API_KEY')

def create_payment(amount, description, customer_email=None):
    response = requests.post(
        'https://api.zynxwallet.com.br/v1/payments/create',
        headers={
            'X-API-Key': api_key,
            'Content-Type': 'application/json'
        },
        json={
            'amount': amount,
            'description': description,
            'customer_email': customer_email,
            'expires_in': 3600,
            'metadata': {
                'created_at': datetime.now().isoformat()
            }
        }
    )

    data = response.json()
    
    if not data['success']:
        raise Exception(data['message'])

    return data['data']

# Usar
payment = create_payment(150.50, 'Compra de produto', 'cliente@example.com')
print(f"Pagamento criado: {payment['id']}")

```

## Campos da Resposta

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `id` | string | ID único do pagamento |
| `amount` | number | Valor do pagamento |
| `currency` | string | Moeda (sempre "BRL") |
| `description` | string | Descrição do pagamento |
| `status` | string | Status do pagamento (pending, completed, cancelled) |
| `qr_code` | string | Código PIX (string) |
| `qr_code_url` | string | URL da imagem do QR Code |
| `pix_key` | string | Chave PIX em formato bruto |
| `customer` | object | Dados do cliente |
| `expires_at` | string | Data de expiração (ISO 8601) |
| `created_at` | string | Data de criação (ISO 8601) |
| `updated_at` | string | Data da última atualização (ISO 8601) |
| `metadata` | object | Dados customizados |

## Códigos de Erro

### 400 - Bad Request

```json
{
  "success": false,
  "error": "invalid_request",
  "message": "O parâmetro 'amount' é obrigatório"
}
```

**Causas comuns:**
- Parâmetros obrigatórios faltando
- Valor inválido (ex: amount negativo)
- Email inválido

### 401 - Unauthorized

```json
{
  "success": false,
  "error": "invalid_api_key",
  "message": "API Key inválida ou expirada"
}
```

**Solução:** Verifique sua API Key.

### 429 - Too Many Requests

```json
{
  "success": false,
  "error": "rate_limit_exceeded",
  "message": "Você excedeu o limite de requisições",
  "retry_after": 60
}
```

**Solução:** Aguarde antes de fazer nova requisição.

## Boas Práticas

<Tip>
  **Dicas para criar pagamentos:**
  - Use `metadata` para rastrear informações customizadas
  - Defina `expires_in` apropriadamente para seu caso de uso
  - Sempre valide o email do cliente
  - Use webhooks para ser notificado quando o pagamento for pago
</Tip>

### Validação de Entrada

```javascript
function validatePaymentInput(amount, description) {
  if (!amount || amount <= 0) {
    throw new Error('Amount deve ser maior que 0');
  }
  
  if (!description || description.length === 0) {
    throw new Error('Description é obrigatória');
  }
  
  if (description.length > 255) {
    throw new Error('Description não pode ter mais de 255 caracteres');
  }
  
  return true;
}
```

### Armazenar ID do Pagamento

Sempre armazene o ID do pagamento para rastreamento:

```javascript
const payment = await createPayment(150.50, 'Compra', 'cliente@example.com');

// Salvar no banco de dados
await db.payments.insert({
  zynx_payment_id: payment.id,
  amount: payment.amount,
  customer_email: payment.customer.email,
  status: payment.status,
  created_at: new Date(payment.created_at)
});
```
