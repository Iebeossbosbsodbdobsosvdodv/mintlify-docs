# Boas Práticas de Segurança

A segurança é fundamental. Siga estas práticas para proteger sua conta Zynx Wallet e seus dados.

## Proteção de API Key

### 1. Nunca Compartilhe sua API Key

Sua API Key é como uma senha. Quem tiver acesso pode fazer operações em sua conta.

<Warning>
  **Nunca:**
  - Compartilhe com outras pessoas
  - Commite no Git ou GitHub
  - Coloque em arquivos de configuração públicos
  - Envie por email ou chat
</Warning>

### 2. Use Variáveis de Ambiente

Sempre use variáveis de ambiente para armazenar sua API Key:

```bash
# .env (não commitar no Git)
ZYNX_API_KEY=sua_api_key_aqui
ZYNX_API_URL=https://api.zynxwallet.com.br
```

```javascript
const apiKey = process.env.ZYNX_API_KEY;
const apiUrl = process.env.ZYNX_API_URL;
```

### 3. Rotacione Regularmente

Troque sua API Key a cada 90 dias:

1. Gere uma nova chave na [Dashboard](https://dashboard.zynxwallet.com.br)
2. Atualize sua aplicação
3. Teste completamente
4. Revogue a chave antiga

### 4. Revogue Imediatamente se Comprometida

Se sua API Key for exposta:

1. Acesse a [Dashboard](https://dashboard.zynxwallet.com.br)
2. Vá para "Configurações" > "API Keys"
3. Clique em "Revogar"
4. Gere uma nova chave
5. Atualize sua aplicação

## Proteção de Webhook

### 1. Validar Assinatura

Sempre valide a assinatura HMAC do webhook:

```javascript
const crypto = require('crypto');

function validateWebhookSignature(payload, signature, secret) {
  const hash = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  
  return hash === signature;
}

app.post('/webhook', express.json(), (req, res) => {
  const signature = req.headers['x-zynx-signature'];
  const secret = process.env.ZYNX_WEBHOOK_SECRET;
  
  if (!validateWebhookSignature(req.body, signature, secret)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }
  
  // Processar webhook
  res.json({ success: true });
});
```

### 2. Usar HTTPS

Sempre configure seu webhook com HTTPS (não HTTP):

```
https://seu-dominio.com/webhook  ✅ Correto
http://seu-dominio.com/webhook   ❌ Inseguro
```

### 3. Guardar Webhook Secret

Seu webhook secret é como uma senha. Guarde em variáveis de ambiente:

```bash
ZYNX_WEBHOOK_SECRET=seu_webhook_secret_aqui
```

## Proteção de Dados

### 1. Usar HTTPS em Sua Aplicação

Sempre use HTTPS para sua aplicação:

```javascript
// Redirecionar HTTP para HTTPS
app.use((req, res, next) => {
  if (req.header('x-forwarded-proto') !== 'https') {
    res.redirect(`https://${req.header('host')}${req.url}`);
  } else {
    next();
  }
});
```

### 2. Validar Entrada

Sempre valide dados do usuário:

```javascript
function validatePaymentInput(amount, description) {
  if (!amount || amount <= 0) {
    throw new Error('Valor inválido');
  }
  
  if (!description || description.length === 0) {
    throw new Error('Descrição obrigatória');
  }
  
  if (description.length > 255) {
    throw new Error('Descrição muito longa');
  }
  
  return true;
}
```

### 3. Sanitizar Dados

Limpe dados antes de armazenar:

```javascript
const sanitizeHtml = require('sanitize-html');

function sanitizeInput(input) {
  return sanitizeHtml(input, {
    allowedTags: [],
    allowedAttributes: {}
  });
}
```

### 4. Usar Prepared Statements

Ao usar banco de dados, use prepared statements para evitar SQL injection:

```javascript
// ❌ Inseguro
const query = `SELECT * FROM users WHERE email = '${email}'`;

// ✅ Seguro
const query = 'SELECT * FROM users WHERE email = ?';
db.query(query, [email]);
```

## Proteção de Conta

### 1. Usar Senha Forte

Sua senha deve ter:

- Mínimo 12 caracteres
- Letras maiúsculas e minúsculas
- Números
- Caracteres especiais

### 2. Ativar Autenticação de Dois Fatores

Na [Dashboard](https://dashboard.zynxwallet.com.br):

1. Vá para "Configurações" > "Segurança"
2. Clique em "Ativar 2FA"
3. Escaneie o código QR com seu app autenticador
4. Insira o código gerado

### 3. Verificar Atividades Suspeitas

Monitore sua conta regularmente:

1. Acesse a [Dashboard](https://dashboard.zynxwallet.com.br)
2. Vá para "Segurança" > "Atividades"
3. Verifique logins e operações recentes
4. Revogue sessões suspeitas

### 4. Manter Contato Atualizado

Mantenha seu email e telefone atualizados para recuperação de conta.

## Proteção de Aplicação

### 1. Usar Rate Limiting

Implemente rate limiting para proteger sua aplicação:

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutos
  max: 100 // limite de 100 requisições por IP
});

app.use('/api/', limiter);
```

### 2. Usar CORS Corretamente

Configure CORS para aceitar apenas domínios confiáveis:

```javascript
const cors = require('cors');

app.use(cors({
  origin: ['https://seu-dominio.com', 'https://app.seu-dominio.com'],
  credentials: true
}));
```

### 3. Usar Headers de Segurança

Adicione headers de segurança:

```javascript
const helmet = require('helmet');

app.use(helmet());

// Ou configure manualmente
app.use((req, res, next) => {
  res.setHeader('X-Content-Type-Options', 'nosniff');
  res.setHeader('X-Frame-Options', 'DENY');
  res.setHeader('X-XSS-Protection', '1; mode=block');
  res.setHeader('Strict-Transport-Security', 'max-age=31536000');
  next();
});
```

### 4. Registrar Atividades

Registre todas as operações importantes:

```javascript
function logActivity(userId, action, details) {
  console.log({
    timestamp: new Date().toISOString(),
    userId,
    action,
    details,
    ip: req.ip
  });
  
  // Salvar no banco de dados
  db.logs.insert({
    user_id: userId,
    action,
    details,
    ip: req.ip,
    created_at: new Date()
  });
}
```

## Checklist de Segurança

- [ ] API Key em variável de ambiente
- [ ] Webhook com HTTPS
- [ ] Assinatura de webhook validada
- [ ] Senha forte na conta
- [ ] 2FA ativado
- [ ] HTTPS em sua aplicação
- [ ] Entrada validada e sanitizada
- [ ] Prepared statements no banco de dados
- [ ] Rate limiting configurado
- [ ] CORS configurado
- [ ] Headers de segurança adicionados
- [ ] Atividades registradas em log

2. **Entre em contato** com [security@zynxwallet.com.br](mailto:security@zynxwallet.com.br)
3. Descreva a vulnerabilidade em detalhes
4. Aguarde nossa resposta

Agradecemos sua ajuda em manter a Zynx Wallet segura!
