# Rate Limiting

A API Zynx Wallet implementa rate limiting para garantir estabilidade e segurança.

## Limites por Tipo de Requisição

| Tipo de Requisição | Limite | Janela |
|-------------------|--------|--------|
| Requisições Gerais | 100 | 1 minuto |
| Criar Pagamento | 50 | 1 minuto |
| Criar Saque | 20 | 1 minuto |
| Login | 10 | 1 minuto |
| Listar Transações | 30 | 1 minuto |

## Headers de Rate Limit

Cada resposta inclui headers informando seu uso:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1644497400
```

### Interpretar os Headers

- **X-RateLimit-Limit**: Limite total de requisições
- **X-RateLimit-Remaining**: Requisições restantes
- **X-RateLimit-Reset**: Timestamp Unix quando o limite será resetado

## Tratando Rate Limit

Quando você excede o limite, recebe um erro 429:

```json
{
  "success": false,
  "error": "rate_limit_exceeded",
  "message": "Você excedeu o limite de requisições. Tente novamente em 60 segundos.",
  "retry_after": 60
}
```

## Implementar Retry com Backoff Exponencial

### JavaScript

```javascript
async function makeRequestWithRetry(url, options, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url, options);
      
      if (response.status === 429) {
        const retryAfter = response.headers.get('Retry-After') || Math.pow(2, i);
        console.log(`Rate limited. Aguardando ${retryAfter}s...`);
        await new Promise(resolve => setTimeout(resolve, retryAfter * 1000));
        continue;
      }
      
      return response;
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      const delay = Math.pow(2, i) * 1000;
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}

// Usar
const response = await makeRequestWithRetry(
  'https://api.zynxwallet.com.br/v1/wallet/balance',
  {
    headers: {
      'Authorization': 'Bearer seu_token'
    }
  }
);
```

### Python

```python
import time
import requests

def make_request_with_retry(url, headers, max_retries=3):
    for i in range(max_retries):
        response = requests.get(url, headers=headers)
        
        if response.status_code == 429:
            retry_after = int(response.headers.get('Retry-After', 2 ** i))
            print(f"Rate limited. Aguardando {retry_after}s...")
            time.sleep(retry_after)
            continue
        
        return response
    
    raise Exception("Max retries exceeded")

# Usar
response = make_request_with_retry(
    'https://api.zynxwallet.com.br/v1/wallet/balance',
    {'Authorization': 'Bearer seu_token'}
)
```

## Boas Práticas

<Tip>
  **Dicas para evitar rate limiting:**
  - Implemente cache quando possível
  - Use webhooks em vez de polling
  - Agrupe requisições quando possível
  - Distribua requisições ao longo do tempo
  - Monitore seu uso de API
</Tip>

### 1. Usar Cache

```javascript
const cache = new Map();
const CACHE_TTL = 60000; // 1 minuto

async function getBalance() {
  const cacheKey = 'balance';
  const cached = cache.get(cacheKey);
  
  if (cached && Date.now() - cached.timestamp < CACHE_TTL) {
    return cached.data;
  }
  
  const response = await fetch('https://api.zynxwallet.com.br/v1/wallet/balance', {
    headers: { 'Authorization': 'Bearer seu_token' }
  });
  
  const data = await response.json();
  cache.set(cacheKey, { data, timestamp: Date.now() });
  
  return data;
}
```

### 2. Usar Webhooks

Em vez de fazer polling a cada minuto, configure webhooks para receber notificações:

```javascript
// Ao invés disso (polling):
setInterval(async () => {
  const response = await fetch('https://api.zynxwallet.com.br/v1/payments/list');
  // ...
}, 60000);

// Faça isso (webhooks):
app.post('/webhook', (req, res) => {
  const event = req.body;
  if (event.type === 'payment.completed') {
    // Processar pagamento
  }
  res.json({ success: true });
});
```

### 3. Agregar Requisições

```javascript
// Ao invés de fazer 100 requisições:
for (let id of paymentIds) {
  await fetch(`/v1/payments/${id}`);
}

// Faça uma única requisição:
const response = await fetch('/v1/payments?ids=' + paymentIds.join(','));
```
